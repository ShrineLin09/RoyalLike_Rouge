# Project Collaboration Notes

## Design Change Workflow

For changes that may affect game design direction, core loop, level structure, economy, progression, roguelite upgrade rules, difficulty, scoring, move limits, or player experience assumptions, do not implement immediately by default.

First reason through the pros, cons, risks, and likely impact. If the change is not an obvious bug fix or an implementation of an already agreed design decision, discuss the recommendation with the user before editing code.

Straightforward bug fixes, compatibility fixes, text/layout fixes, and implementation tasks that follow an already agreed direction can still be executed directly.
